# Структура

| Часть | Где лежит | Что это |
| --- | --- | --- |
| Landing | В корне: `index.html`, `assets/`, `styles/`, `scripts/` | страница |
| SPA | `frontend/` | продуктовый интерфейс |
| API | `backend/` | сервер + база данных |

```text
├── index.html
├── styles/                 # Landing CSS
├── scripts/                # Landing JS
│
├── assets/
│
├── backend/                # API
│   └── app/
│       ├── core/           # Конфиг, БД, безопасность и т.д.
│       ├── modules/        # Бизнес-модули
│       └── api/            # Руты, схемы запросов/ответов, зависимости
│
├── frontend/               # SPA
|
├── infra/                  # docker-compose, деплой, шаблоны переменных окружения
|
├── docs/
├── README.md
|
└── .github/workflows/      # CI
```

## Из чего состоит бэкенд

В планах разделение:

- `identity` — пользователи, группы, участники, роли и права. Кто что может видеть и редактировать. Коллекция принадлежит пользователю или группе.
- `ingestion` — приём конспектов: загрузка файла или регистрация URL (который потом сам обновляется). Здесь же проверки на мусор (тип файла, размер, число страниц), защита от дублей и контроль квот на входе.
- `documents` — хранилище: ссылки на сырые файлы (PDF, картинки) + структурированная модель документа + версии (обновился URL — появилась новая версия).
- `parsing` — превращение PDF в структуру.
- `knowledge` — поиск: по ключевым словам (полнотекстовый) и по смыслу (векторный). Сюда же — поиск определения по всем конспектам.
- `study` — учебные материалы: карточки, колоды, тесты/квизы.
- `billing` — токеномика.

### Конвейер парсинга

1. `Extract` — вытащить текст, картинки и формулы.
2. `Segment` — разбить на логические блоки по структуре (заголовки, нумерация, ключевые слова).
3. `Merge` — по контексту склеить блоки, собрать дерево.
4. `Index` — записать представления для поиска.
